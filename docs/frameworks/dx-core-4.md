# DX Core 4

## 概要

DORAの著者Nicole Forsgren、SPACEの著者Margaret-Anne StoreyとThomas Zimmermanらがアドバイザーとして参加し、Laura Tacho（DX CTO）とAbi Noda（DX CEO）が開発した統合フレームワーク。DORA + SPACEを統合し、AI時代を前提に設計されている。

---

## 4つの次元

| 次元 | 説明 |
|------|------|
| **Speed（速度）** | チームがどれだけ迅速にソフトウェアを提供するか |
| **Effectiveness（有効性）** | 価値を生む活動に時間が使われているか |
| **Quality（品質）** | 成果物の信頼性と安定性 |
| **Business Impact（ビジネスインパクト）** | 達成された組織的成果 |

## 設計原則

- **多次元アプローチ** — 速度のために品質を犠牲にする等のトレードオフを防止
- **組織全体への適用性** — 経営層からフロントラインチームまで
- **迅速な展開** — 数ヶ月ではなく数週間でセットアップ可能
- **Developer Experience Indexによるバランス**

## 実績

300以上の組織でテスト・改善され：
- エンジニアリング効率 3-12%向上
- 機能開発に費やすR&D時間 14%増加

## 広木大地氏の3階層モデルとの対応

| DX Core 4 | 広木氏の3階層 | 説明 |
|-----------|-------------|------|
| Speed | 仕事量生産性 | 作業のスループット |
| Effectiveness | 期待付加価値の生産性 | 価値ある活動に時間を使えているか |
| Quality | （横断的） | 仕事量〜実現付加価値の各層の品質 |
| Business Impact | 実現付加価値の生産性 | 実際のビジネス成果 |
